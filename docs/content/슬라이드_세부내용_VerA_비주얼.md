## Slide 3. DX에서 AX로 — 패러다임 전환

**레이아웃**: 좌우 대비 구조

**헤드라인**: 디지털 전환(DX)을 넘어, AI 전환(AX) 시대

**좌측 — DX 시대** (회색 톤):
- 자동화(Automation)
- 수동 운영 → 디지털 운영
- 사람이 분석하고 결정

**우측 — AX 시대** (파란색/밝은 톤):
- 자율화(Autonomy)
- AI가 분석, 예측, 실행
- 매장이 스스로 최적화

**인용**:
> "DX가 자동화(Automation)라면, AX는 자율화(Autonomy)이다" — 사티아 나델라, Microsoft CEO

**하단 키워드 4개**:
1. 에이전틱 서비스 (Agentic SaaS) — 매장 내 AI 에이전트
2. 산업 특화 AI (Vertical AX) — 범용 LLM → 리테일 SLM
3. AI 거버넌스/보안 — 온프레미스, 데이터 주권
4. AI Build-in HW — On-Device AI, 3단 분산처리

## Slide 7. 리테일 전문 AI — 3대 핵심 역할

**레이아웃**: 3-pillar 인포그래픽

**헤드라인**: AIsirius AI가 매장에서 하는 3가지

**Pillar 1: AI 콘텐츠 생성**
- 상품명만 입력 → 이미지/동영상 **즉시** 생성
- 프로모션 배너, 이벤트 이미지 자동 제작
- 매장 디바이스에 즉시 배포
- *기존: 외주 제작 5일 → AIsirius: 1분*

**Pillar 2: AI 매장 분석**
- POS/ERP 데이터 실시간 연동
- 플래노그램 기반 진열 현황 분석
- 고객 행동/동선/시선 AI 분석 (Vision AI)
- *블랙박스였던 오프라인 매장 데이터를 가시화*

**Pillar 3: AI 자율 운영**
- 다이나믹 프라이싱 (재고/시간 기반 자동 가격 조정)
- 진열 최적화 AI 제안
- 센서 기반 실시간 의사결정
- *사람 개입 최소화 → AX(AI Transformation) 실현*

## Slide 8. 전용 AI vs 범용 AI

**레이아웃**: 좌우 대비 테이블 + 하단 강조 박스

**헤드라인**: 범용 AI가 못하는 것, 전용 AI는 합니다

**비교 테이블**:
| 항목 | 범용 AI (Midjourney, DALL-E) | AIsirius 전용 AI |
|------|---------------------------|-----------------|
| **저작권** | 제품 이미지 그대로 생성 **불가** (저작권 이슈) | 매장 사용권 → 제품 이미지/동영상 **그대로 사용** |
| **브랜드 정확도** | 로고/색상 할루시네이션 | LoRA 파인튜닝 → **100% 보장** |
| **이미지 비율** | 1:1, 16:9 등 정형만 | 1920x158 등 **비정형 자유 생성** |
| **해상도** | 1024x1024 고정 | stretched LCD **모든 해상도** 대응 |
| **동영상** | 길이/시간 제한 | **제한 없음**, 매장용 최적화 |
| **리테일 특화** | 범용 학습 | **리테일 최적** 이미지/동영상 |
| **클라우드 의존** | 100% 외부 (데이터 유출 위험) | **자체 서버** (온프레미스, 데이터 주권) |
| **학습/튜닝** | 불가 (API 사용만) | **지속적 학습/파인튜닝** 가능 |

**하단 강조 박스**:
> AIsirius는 이를 위해 **회사 자체 전용 AI 서버**를 구축하여 운영 중
> (ComfyUI 기반, Z-image / Wan / Flux.2 / Qwen 통합)

**시각 자료**: 좌측에 범용 AI가 생성한 왜곡된 제품 이미지 vs 우측에 AIsirius가 생성한 정확한 제품 이미지 대비 예시

## Slide 9. AI 콘텐츠 자동 생성 — 데모

**레이아웃**: 4단계 프로세스 + 실제 생성물 스크린샷

**헤드라인**: 상품명 하나로, 전문 스튜디오급 콘텐츠를 **1분 만에**

**4단계 프로세스**:
```
① 한국어 입력          ② AI 프롬프트 엔지니어링     ③ AI 생성              ④ 매장 즉시 적용
"바게트, 나무 쟁반"  →  최적 프롬프트 자동 변환   →  이미지/동영상 생성  →  디바이스 자동 반영
```

**실제 시연 예시** (2026년 2월 인탑스 시연):
- AI 이미지: "HAPPY NEW YEAR, 일출, red horse, 따뜻한 톤" → 이벤트 배너
- AI 이미지: "SD 3D 피규어, 젊은 한국 남자, 아이돌 의상" → 캐릭터
- AI 동영상: "잘 구워진 바게트, 나무 쟁반" 이미지 3장 → **7.5초 동영상**
- 생성된 동영상을 [존쿡브레드 보르첸 바게트] 상품에 적용 → 29인치 디바이스에 즉시 반영

**하단 스펙**:
- 이미지 생성: 다양한 크기/비율 자유 (800x800, 1920x158, 3840x158 등)
- 동영상 생성: 이미지 기반 7.5초~ 동영상 변환
- 브랜드 LoRA: 이마트, 롯데마트 등 브랜드별 전용 학습 모델

## Slide 10. AI 매장 분석 & 제안

**레이아웃**: 데이터 흐름도 + 핵심 지표

**헤드라인**: POS 데이터 + AI = 매장의 블랙박스를 여는 열쇠

**데이터 흐름도**:
```
┌──────────┐    ┌──────────┐    ┌──────────────┐    ┌─────────────┐
│ POS/ERP  │ →  │ AI 분석  │ →  │ 플래노그램   │ →  │ AI 제안     │
│ 매출 데이터│    │ 패턴 인식 │    │ 진열 현황 매칭│    │ 실행 자동화  │
└──────────┘    └──────────┘    └──────────────┘    └─────────────┘
       │                                                    │
   센서/비전 ─── 고객 행동 분석 ── 시선/동선/체류시간 ─── 콘텐츠 최적화
```

**AI 분석 결과물**:
- 상품별 매출/체류시간 히트맵
- 진열 위치 vs 매출 상관관계
- 시간대별 고객 유형 분석
- 재고 소진 예측 → 자동 가격 조정

**핵심 메시지**:
> 온라인은 클릭/전환/이탈률을 실시간 추적
> 오프라인은 **블랙박스** → AIsirius AI가 이 데이터를 가시화

## Slide 11. 3단 분산 AI 시스템 — 특허 기술

**레이아웃**: 3계층 아키텍처 다이어그램 + 우측 시나리오

**헤드라인**: 세계 최초, 매장용 3단계 분산 AI 처리 시스템
**서브**: 특허 출원 10-2025-0208402

**아키텍처 다이어그램**:
```
┌──────────────────────────────────────────────────┐
│ 제3계층: AI 서버                                   │
│  • LLM / 강화학습 기반 고정밀 추론                   │
│  • 개인화 추천, 장기 프로파일링, 트렌드 분석           │
│  • 경량 모델 생성 → 엣지/AP 배포                     │
├──────────────────────────────────────────────────┤
│ 제2계층: AI 액세스 포인트 (AP)                       │
│  • 복수 엣지 이벤트 수집/분류                         │
│  • 지역 DB + 규칙 기반 → 중간 판단                   │
│  • 추천/비교/위치 안내 처리                           │
├──────────────────────────────────────────────────┤
│ 제1계층: AI 엣지 디바이스 (AIDR)                     │
│  • 멀티센서 입력 (카메라/마이크/근접/조도)              │
│  • 경량 CNN/RNN → 입력 난이도 자율 판단               │
│  • 저난이도: 즉시 응답 (0.1초)                       │
└──────────────────────────────────────────────────┘
```

**우측 — 실제 시나리오**:
| 상황 | 처리 계층 | 응답 시간 |
|------|---------|----------|
| 고객 접근 → 화면 활성화 | 엣지 | 0.1초 |
| "이 상품 가격?" | 엣지 | 즉시 |
| "비슷한 상품 추천해줘" | AP | 1~2초 |
| "지난달 구매 기반 추천" | 서버 | 2~3초 |

**핵심 차별점**:
- 입력 복잡도에 따라 **각 계층이 자율적으로 처리 주체 결정** (자율 분기 로직)
- 서버 의존도 최소화 → **네트워크 비용/부하 80%+ 절감**
- 서버 학습 모델을 엣지로 **지속 배포** → 시스템이 진화

**추가 등록 특허**: 배터리 충전 장치 2건 (10-2303197, 10-2331690)

## Slide 12. 리테일 미디어 플랫폼 — RMN+ESL 통합

**레이아웃**: 플랫폼 아키텍처 다이어그램

**헤드라인**: 하나의 플랫폼으로, 매장의 모든 디스플레이를 통합 관리

**아키텍처 4-Layer**:
```
┌──────────────────────────────────────────────┐
│  Layer 4: AI 레이어                            │
│  AI 콘텐츠 생성 | POS 분석 | 자율 운영 제안     │
├──────────────────────────────────────────────┤
│  Layer 3: 플랫폼 레이어                         │
│  CMS 통합관리 | RMN 광고 | ESL 가격 관리       │
├──────────────────────────────────────────────┤
│  Layer 2: 디바이스 레이어                        │
│  LCD(Android) | EPD(전자잉크) | 기존 기기       │
│  모든 사이즈 + stretched 비정형                  │
├──────────────────────────────────────────────┤
│  Layer 1: HW 소싱 레이어                        │
│  중국 파트너사 소싱 | 합납품 | VESA 맞춤 설계    │
└──────────────────────────────────────────────┘
```

**RMN이 제공하는 5가지**:
1. 가격 표시 (기존 ESL 기능 내장)
2. 상품 추가 정보 (스펙, 리뷰 등)
3. 이미지/동영상 콘텐츠
4. 디지털 광고 (CPG 브랜드 → 선반 광고 구매)
5. 고객 인터랙션 (터치, QR, 게이미피케이션)
