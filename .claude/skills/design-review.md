# 디자인 리뷰 스킬

생성된 프레젠테이션의 디자인 품질을 평가하고 개선 제안을 합니다.

## 사용법
`/design-review [파일경로]`

## 평가 기준 (10점 만점)

### 1. 시각적 계층 (Visual Hierarchy)
- 제목이 본문보다 2배 이상 큰가?
- 핵심 정보가 시선 흐름(Z/F패턴)에 맞는가?
- 컬러로 중요도 구분이 되는가?

### 2. 인지 부하 (Cognitive Load)
- 슬라이드당 1개 핵심 메시지인가?
- 불릿 4개 이하인가?
- 텍스트 7줄 이하인가?

### 3. 브랜드 일관성 (Brand Consistency)
- AIsirius 브랜드 컬러 준수?
- 폰트 가이드라인 준수?
- 크롬 요소(헤더/푸터/로고) 일관성?

### 4. 접근성 (Accessibility)
- 텍스트-배경 대비 4.5:1 이상?
- 최소 폰트 크기 준수?
- 색상만으로 정보 전달하지 않는가?

### 5. 콘텐츠 정합성
- Ver.A/B 내용 일치?
- 시장 데이터 최신?
- 오탈자/표기 통일?

## 동작
1. PPTX → 텍스트 추출 (python-pptx 또는 markitdown)
2. 슬라이드별 요소 분석 (텍스트량, 이미지, 차트)
3. 기준별 점수 산출
4. 개선 우선순위 제안
5. 결과를 `docs/analysis/` 저장
