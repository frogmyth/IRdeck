# 프레젠테이션 작성 규칙

## 핵심 원칙
- **청중 중심 설계**: 투자자/파트너 관점에서 가치 전달
- **Assertion-Evidence 구조**: 주장(제목) + 증거(비주얼/데이터)로 구성
- **슬라이드당 1개 핵심 메시지**: 과도한 정보 금지

## Ver.A / Ver.B 관계
- **동일한 내용**, 표현 방식만 다름 (둘 다 외부 전달용)
- Ver.A: 이미지/인포그래픽 중심, 텍스트 간결
- Ver.B: 데이터/표/텍스트 중심, 상세 설명
- 내용 업데이트 시 양쪽 버전 모두 반영 필수

## 콘텐츠 계층 (CEO 확인)
1. **AI 핵심 역량** — 리테일 전문 AI 기업 포지셔닝
2. **플랫폼 통합** — RMN + ESL 통합 미디어 플랫폼
3. **시장 기회** — ESL/RMN/AI 시장 성장 데이터
4. **비즈니스 모델** — 수익 구조 (SaaS + 광고 + HW)
5. **차별점** — 전용 AI vs 범용 AI

## 슬라이드 디자인 가이드
### 텍스트
- 제목: 40pt 이상 (대형 화면 발표 기준)
- 본문: 24pt 이상 (Ver.A), 14pt 이상 (Ver.B)
- 한 슬라이드에 문장 7줄 이하
- 불릿 사용 시 4개 이하

### 비주얼
- 이미지/인포그래픽: 슬라이드 면적의 50% 이상 (Ver.A)
- 차트: 데이터 라벨 직접 표시 (범례 최소화)
- 테이블: 5행 이하 권장, 핵심 셀 색상 강조

### 레이아웃 타입 (18종)
- TITLE_COVER, TWO_COLUMN, CHART_SLIDE, CHART_DETAILED
- TEXT_HEAVY_BULLETS, COMPARISON_TABLE, THREE_PILLAR
- NEWS_QUOTE, INFOGRAPHIC_NUMBERS, TEXT_HEAVY_TABLE
- THANK_YOU, GENERIC 등

## 텍스트 상자 규칙 (python-pptx)
- **자동 맞춤**: `tf.auto_size = MSO_AUTO_SIZE.SHAPE_TO_FIT_TEXT` (도형을 텍스트 크기에 맞춤)
- **자동 줄바꿈 OFF**: `tf.word_wrap = False` (도형의 텍스트 배치 끄기)
- **줄바꿈**: 명시적 Enter(새 문단)으로만 처리. `_wrap_text_to_lines()` 함수로 박스 너비에 맞게 자동 줄바꿈
- 모든 텍스트박스에 위 설정을 기본으로 적용

## 이미지 규칙
- **가로세로 비율 변경 금지**: 이미지의 원본 비율을 절대 변경하지 않음
- **배치 순서**: 축소/확대로 영역에 맞춤 → 안 되면 크롭하여 배치
- **이미지 소스 우선순위**:
  1. `assets/images/slides/` (프로젝트 내 정리된 이미지)
  2. 기존 CEO PPT에서 추출한 이미지 (`from_ppt/`)
  3. 회사소개서 이미지 폴더 (`G:/00.googledrive/07.AIsirius/20.회사소개서/images/`)
  4. 나노바나나2로 신규 생성
- 인포그래픽 텍스트 → python-pptx 텍스트 오버레이 (수정 가능)
- 원근감 적용 텍스트 → 이미지 내 포함 (나노바나나2)
- 제품 사진 → 전용 AI로 생성 (저작권 안전)

## 파일 버전 관리
- **오류 수정/설정 변경**: `--overwrite` 옵션으로 최신 파일 덮어쓰기 (새 파일 안 만듦)
- **내용/디자인 변경**: 새 순번 파일 생성 (`_YYYYMMDD_NN.pptx`)
- **Ver.A와 Ver.B는 별도 순번**으로 관리
- 기존 파일은 절대 삭제하지 않음

## 기밀 정보 관리
- `general` 버전: 투자 조건, 주식 현황 제외
- `[회사명]` 버전: 해당 회사 맞춤 + 워터마크
- credentials.md는 git에서 제외
